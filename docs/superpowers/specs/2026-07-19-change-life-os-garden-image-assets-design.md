# Change-Life OS 花園 GPT 圖像素材升級設計

## 目標

以 GPT 生成嘅原創圖像素材取代現時由 HTML 元素及 CSS 幾何形狀拼成嘅花園、四維象徵及守護靈。保留既有養成規則、四維數值、主題預覽及無障礙語意，但令畫面呈現成一個一致、值得收藏嘅遊戲場景。

## 已選方向

採用分層圖像素材。美術為溫暖日系繪本 × 精緻奇幻花園，角色保持中性、無明確宗教身份，沿用深綠 `#143C2D`、橙 `#DA8134`、深橙 `#9A3D1A`、白及金色。畫面唔加入文字、商標、經文、寫實人物或現成 IP 風格。

## 素材結構

- 兩張 16:10 花園底圖：免費黃昏花園及「初光之園」晨光版本。
- 四組透明成長素材：心智為靜水與書頁、身體為根系與植物、精神為風與微光、天職為道路與火種。
- 每個維度有四級視覺：休眠、萌芽、成長、盛放；網站按永久成長值選擇相應圖片。
- 守護靈有五個圖像狀態，對應現有階段 0–4；輪廓、服裝及視角保持一致，只增加細節、姿態自信及光環。
- 支持者主題額外使用晨光、金色水面、粒子及光環圖層，但唔改變任何玩法數值。

生成先建立一張風格基準圖，再以該圖作其餘素材嘅視覺參考。透明素材若由生成結果帶有純色背景，只作確定性去背、裁切、縮放及壓縮，唔以程式重新繪製主體。

## 介面及狀態映射

`GrowthGarden` 會改用真正嘅 `<img>`／`picture` 素材層。HTML 只負責容器、說明文字、數值及互動；唔再用 `border-radius`、`clip-path`、陰影組合或偽元素繪製景物同角色。

維度成長級別固定為：0 粒＝休眠、1–2 粒＝萌芽、3–4 粒＝成長、5 粒或以上＝盛放。守護靈繼續直接使用現有 0–4 階段。切換素材只做短暫透明度淡入；`prefers-reduced-motion` 下立即切換。

每層使用固定百分比定位，桌面、平板及 390px 手機共享同一場景構圖。圖像屬裝飾時使用空 `alt`；整個花園仍由 `figure` 嘅本地化標籤，以及四維名稱、意象同數值提供螢幕閱讀器資訊。

## 完成卡

7／14 完成卡會重用新花園底圖及當刻四維／守護靈素材，透過本機 Canvas 合成下載。卡片仍只顯示完成數、週期日期、產品版本及花園視覺，唔加入四維分數、任務或復盤內容。

## 效能及故障處理

- 首屏底圖使用響應式 WebP；透明圖層優先使用 WebP 或 PNG，保持可讀畫質並控制檔案體積。
- 只預載當前主題及當前階段；預覽支持者主題時先載入相應底圖。
- 圖像載入失敗時保留可讀嘅維度標籤及數值，唔回退到舊 CSS 圖案。
- 所有生成原圖保留喺 `outputs/`；網站用嘅裁切及壓縮版本放入 `public/garden/`。

## 驗證

- 單元測試覆蓋成長值到四級素材嘅邊界，以及守護靈 0–4 路徑。
- 元件測試確認場景使用圖像素材，舊 `.garden-motif`、`.guardian-head`、`.guardian-body` 等繪圖節點不再存在。
- 390px、平板及桌面視覺 QA；檢查免費／支持者預覽、每級成長、圖片失敗、鍵盤、螢幕閱讀器及 reduced motion。
- 完成卡私隱測試保持通過，並確認 Canvas 輸出包含新圖像。
- Production build、現有單元測試及 Pages 合約測試全部通過後先部署。
