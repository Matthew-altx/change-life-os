# Change-Life OS 手機花園 UI 精修設計

## 目標

改善現有手機花園畫面嘅視覺層級，令 GPT 生成嘅花園插畫重新成為主角，同時保留四維成長及守護靈進度嘅即時可讀性。

本改動只處理 UI、排版及現有圖像資產嘅顯示方式；不修改玩法、種子規則、成長速度、付費權益、儲存格式或文案語意。

## 已批准方向

採用「沉浸式輕 HUD」：守護靈直接浮於花園中央，四維資訊縮成較輕量嘅圖像卡，種子資訊改為頂部橫向 HUD。手機畫面減少約一成垂直高度，避免花園佔滿首屏並改善固定底部導航附近嘅閱讀節奏。

## 介面結構

### 守護靈

- 從五張現有守護靈 WebP 產生透明背景版本，移除深綠色膠囊框。
- 守護靈以獨立圖像直接置於花園中央，不再顯示實色容器、外框或大型底板。
- 五個成長階段繼續由 `guardianStage` 0–4 選圖。
- 第 4 階段可保留非常輕微呼吸動畫；`prefers-reduced-motion` 時停用。
- 圖像仍為裝飾性內容，現有守護靈階段 `aria-label` 保持不變。

### 頂部 HUD

- 目前左上角大型種子方盒改為單行膠囊：`生命種子 {total} · 守護靈 {stage}/4`。
- 手機置於花園頂部中央，桌面可維持靠左但縮細。
- 初光主題嘅支持者徽記保留於右上角，不與 HUD 重疊。

### 四維卡

- 維持心智／身體置頂、精神／天職置底嘅 2×2 空間關係。
- 手機卡片高度由 108px 降至約 88–94px，左右間距維持最少 10px。
- 圖像遮罩減淡，讓水面、根系、風、道路細節更清楚。
- 標題保持高對比；手機隱藏副標籤，桌面保留。
- 數字圓章縮細至約 24px；進度線維持 3px，以金色表示已成長比例。
- 成長值仍以現有 `gardenGrowthLevel` 計算，不改數據。

### 手機尺寸與安全區

- `<=460px` 時花園高度由 520px 調整至約 470px。
- 守護靈移除容器後以實際角色大小定位，避免與上下四維卡相撞。
- 底部四維卡提升少量，花園後方內容與固定 mobile nav 之間保留清晰分段。
- iOS Safari 390px 闊度為主要驗收尺寸；同時檢查 360px 及 430px。

## 圖像處理

- 只對現有原創守護靈素材做確定性去背，不再生成新圖。
- 去背以深綠背景色及邊緣連通區域為依據，保留角色內部深色面部、眼睛、金色光暈及粒子。
- 輸出 WebP 需保留 alpha，單張目標寬度約 300px。
- 若自動去背會侵蝕角色輪廓，改用柔邊遮罩而非改回實色膠囊框。

## 響應式及無障礙

- 桌面版只縮減 HUD 重量及移除守護靈底板，不改雙欄頁面架構。
- 四維文字、數值及守護靈階段繼續存在於可存取樹；背景圖維持空 alt。
- 所有顏色維持既定深綠、橙、深橙、白及金色組合。
- 鍵盤操作不受影響；本改動不新增互動控制。

## 驗收

- 390×844 iOS Safari 視覺：花園插畫係第一視覺，守護靈冇深綠底板，四維卡不遮住角色。
- 360px、430px、平板及桌面無溢出、重疊或文字截斷。
- 免費／初光背景均清楚，支持者徽記不與 HUD 重疊。
- 第 0、1、4 階段守護靈可辨識；完成一粒種子後正確換圖。
- `prefers-reduced-motion` 停止守護靈動畫。
- `npm run test:pages` 全部通過，瀏覽器 console 無錯誤。

## 非目標

- 不修改每日 check-in、任務、種子、週期或 XP 邏輯。
- 不開啟 Stripe 或 Cloudflare Worker 生產付款。
- 不重新設計全站導航、行動卡或支持者商店。
- 不加入新動畫框架、分析追蹤或新圖像生成服務。
